========================================================================
         TECHNICAL IMPLEMENTATION DETAILS - HR DATA PIPELINE
========================================================================

1. TECHNICAL ENVIRONMENT & STACK
--------------------------------
The data analysis pipeline is built using the standard Python PyData stack:
- Python 3.x
- Pandas: Handles data cleaning, slicing, grouping, and CSV I/O.
- NumPy: Performs numerical operations and computes correlations.
- Matplotlib (pyplot): Serves as the canvas manager for chart renderings.
- Seaborn: Creates high-level statistical plots with custom palettes.
- Jupyter Notebook format: .ipynb (nbformat 4.0).

2. INPUT AND OUTPUT FILES
-------------------------
- Raw Input File:       /content/HR_Dataset.csv
- Cleaned Intermediate:  cleaned_data_HR.csv (exported at cell 18)
- Secondary Input:      /content/cleaned_data_HR.csv (re-loaded at cell 19)

3. STEP-BY-STEP DATAFLOW LIFECYCLE
----------------------------------
The notebook execution flows sequentially as follows:

[Cells 1 - 2]   Environment setup and data loading. Reads the raw CSV.
[Cells 3 - 7]   Initial metadata audit (shape, column names, types, summary statistics, raw duplicate counts).
[Cell 8]        Initial date conversion:
                df['StartDate'] = pd.to_datetime(df['StartDate'], format='%d-%b-%y', errors='coerce')
                Also parses ExitDate, DOB, Survey Date, and Training Date.
[Cells 9 - 10]  Computes missing percentage using:
                (df.isna().sum() / len(df) * 100).sort_values(ascending=False)
[Cell 11]       Prunes the 'Survey Date' column as it has >99% null rate.
[Cells 12 - 15] Post-prune null checks and duplicate checks.
[Cell 16]       De-duplication: df = df.drop_duplicates()
[Cells 17 - 18] Verifies duplication removal (expects 0) and writes to cleaned_data_HR.csv.
[Cells 19 - 22] Verification of the exported CSV file. Loads the exported CSV.
                Re-converts columns back to datetime objects since CSV format
                stores datetime columns as raw text objects.
[Cells 23 - 25] Categorical value-count aggregates for BusinessUnit and EmployeeStatus.
[Cells 26 - 29] Metrics baselines (mean performance, max performance, mean engagement, training cost statistics).
[Cells 30 - 33] Groupby aggregations for:
                - Performance Rating by Department
                - Satisfaction Score by Business Unit
                - Training Cost by Training Program Name
                - Work-Life Balance Score by Gender
[Cells 34 - 39] Graphical visualisations:
                - Don't Chart: Visualizes department percentages using plt.pie() with width=0.3.
                - Seaborn Countplot: Visualizes department headcount with x-tick labels rotated by 45 degrees.
                - Seaborn Histplot: Displays employee ratings with kernel density curve overlays.
                - Styled Histplot: An aesthetically polished performance histogram with customized slate blue bars, coral KDE, and despined axis borders.
                - Boxplot: Visualizes training program costs, displaying outliers and box spreads.
                - Grouped Countplot: Displays EmployeeStatus by BusinessUnit with counts displayed directly on the bar labels.
[Cells 40 - 44] Key Business Queries:
                - Finding the largest BusinessUnit: df['BusinessUnit'].value_counts().idxmax()
                - Ranking departments: df.groupby('DepartmentType')['Current Employee Rating'].mean().sort_values(ascending=False)
                - Correlation coefficients using Pandas' series correlation:
                  df['Engagement Score'].corr(df['Satisfaction Score'])
                  df['Training Duration(Days)'].corr(df['Current Employee Rating'])
                - Outlier extraction: df.loc[df['Training Cost'].idxmax()]

4. SYSTEM ROBUSTNESS & EXTENSIBILITY
------------------------------------
- String Date Formats: Handled custom format `%d-%b-%y` (e.g., 20-Sep-19) and `%d-%m-%Y` (e.g., 20-09-1990) with error coersion.
- De-spining in Seaborn: sns.despine(left=True, bottom=True) removes visual noise from plots to maintain clean visual design.
- Re-importing Preprocessing: The re-import verification ensures the output file matches expectations and preserves integrity when read back by other pipelines.
